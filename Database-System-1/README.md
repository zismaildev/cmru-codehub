## 🎯 เกี่ยวกับรายวิชา
เข้าใจหลักการออกแบบฐานข้อมูลเชิงสัมพันธ์ **(Relational Database)** ตั้งแต่การวิเคราะห์ความต้องการ การออกแบบ Schema (ER Diagram) ไปจนถึงการเขียนคำสั่ง **SQL** เพื่อจัดการข้อมูลอย่างมีประสิทธิภาพ

## 🔑 สิ่งที่เรียนรู้ (Key Learnings)
- **Database Design:** การออกแบบ ER Diagram และการแปลงเป็น Table
- **Normalization:** เทคนิคการลดความซ้ำซ้อนของข้อมูล (1NF, 2NF, 3NF) เพื่อความถูกต้อง (Integrity)
- **SQL Queries:**
  - **DDL**: `CREATE`, `ALTER`, `DROP`
  - **DML**: `SELECT`, `INSERT`, `UPDATE`, `DELETE`
  - **Complex Queries**: `JOIN`, `GROUP BY`, `HAVING`

## 📂 โครงสร้างภายใน (Repository Structure)
- **`lab/`**: ไฟล์ `.sql` รวบรวมคำสั่ง SQL ที่ฝึกเขียนในคาบเรียน (e.g., Workshop สร้างตาราง, Workshop ดึงข้อมูล)
