## 📦 Project Overview

โปรเจกต์นี้คือ GitHub Repository ชื่อ **`cmru-codehub`**\
ใช้เป็น **Portfolio ทางการศึกษา** ของนักศึกษาวิทยาการคอมพิวเตอร์

เป้าหมายคือจัดโครงสร้าง repository ให้:

> 👉 แสดงชัดเจนว่าเจ้าของ repo เรียน **รายวิชาอะไรบ้าง**\
> 👉 และได้ **ทักษะอะไรจากแต่ละวิชา**

------------------------------------------------------------------------

## 🎯 Project Goals

-   ทำให้ repository นี้อ่านแล้วเข้าใจง่ายภายใน **1 นาที**
-   ใช้เป็น Portfolio สำหรับ **อาจารย์** และ **HR / Recruiter**
-   แสดงพัฒนาการและพื้นฐานด้าน **Computer Science**
-   รองรับ:
    -   โค้ดเก่า
    -   lab
    -   assignment
    -   mini project

------------------------------------------------------------------------

## ❗ Core Rules (ต้องปฏิบัติตาม)

-   ❌ ห้ามจัดโครงสร้างตาม **ชั้นปี**
-   ✅ ต้องจัดโครงสร้างตาม **รายวิชา (Course-based)** เท่านั้น
-   แต่ละรายวิชาต้องเป็นโฟลเดอร์แยก และมี `README.md`
-   ❌ ไม่ต้อง refactor หรือแก้ logic โค้ด
-   ✅ ให้ความสำคัญกับ **คำอธิบายสิ่งที่เรียนรู้** มากกว่า code quality

## 📝 README Writing Rules

### README หลัก (Root)

ต้องอธิบาย: - จุดประสงค์ของ repository -
แนวคิดการจัดโครงสร้างตามรายวิชา - รายวิชาที่มีทั้งหมด - เทคโนโลยี / ภาษา
ที่เกี่ยวข้อง

### README รายวิชา

ต้องอธิบาย: - วิชานี้เรียนเกี่ยวกับอะไร - หัวข้อสำคัญที่เรียน - ภาษา /
เครื่องมือที่ใช้ - สิ่งที่ได้เรียนรู้ - หมายเหตุเกี่ยวกับโค้ดเก่า
(ถ้ามี)

> ใช้ภาษาทางการ กระชับ เข้าใจง่าย\
> ❌ ไม่ใช้ภาษาการตลาด
